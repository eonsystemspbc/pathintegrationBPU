# Optic-Lobe Optic-Flow Benchmark

This benchmark tests whether the fly optic-lobe connectome provides a useful
recurrent initialization for a visual ego-motion task.

The task is inspired by connectome-constrained fly visual-system modelling,
including Lappalainen et al., Nature 2024, where optic-lobe connectome structure
was used to constrain recurrent networks trained on visual motion/optic-flow
objectives.

## Task

Each training example is a short movie rendered onto a hexagonal sensor lattice.

The input sequence is generated by:

1. Procedurally creating a panoramic texture.
2. Sampling it through a fly-like hex lattice.
3. Applying known ego-motion:
   - yaw rotation,
   - forward translation,
   - lateral translation.
4. Rendering the transformed panorama through acceptance-angle blur.
5. Applying a difficulty-controlled contrast/noise/sample-size ladder.

The model receives a tensor:

```text
[batch, timesteps, hex_lattice_samples]
```

and predicts:

```text
[yaw_rate, forward_velocity, lateral_velocity]
```

at every timestep.

## Models

All models are size matched:

- same number of recurrent neurons,
- same number of trainable recurrent edge parameters,
- same input layer shape,
- same readout shape,
- same optimizer/training schedule.

The compared models are:

### `optic_lobe_seeded`

The recurrent graph starts from the prepared FlyWire optic-lobe adjacency:

```text
support = optic-lobe connectome edges
weights = scaled optic-lobe synapse weights
```

The recurrent edge values are trainable.

### `shuffled_topology`

The model preserves:

```text
same neuron count
same edge count
same recurrent weight multiset
```

but randomizes the recurrent support/topology.

This asks whether the specific optic-lobe wiring arrangement matters.

### `random_sparse`

The model preserves:

```text
same neuron count
same edge count
same trainable parameter count
```

but uses random sparse support and Gaussian recurrent weights.

This asks whether any size-matched sparse recurrent network can solve the task.

## AWS Run

From the repo root on the AWS machine:

```bash
cd /home/ec2-user/pathintegrationBPU/pathintegrationBPU
git pull origin main
source /home/ec2-user/pathintegrationBPU/.venv/bin/activate

OUT=/home/ec2-user/pathintegrationBPU/pathintegrationBPU/outputs/flywire_optic_lobe_flow_medium_3seed
mkdir -p "$OUT"

python scripts/run_optic_flow_benchmark.py \
  --mode all \
  --output-dir "$OUT" \
  --cache-dir "$OUT" \
  --flywire-download-dir /home/ec2-user/pathintegrationBPU/flywire_cache \
  --device cuda \
  --difficulty medium \
  --models optic_lobe_seeded shuffled_topology random_sparse \
  --seeds 0 1 2 \
  --epochs 30 \
  --patience 8 \
  --batch-size 64 \
  --log-every-seconds 30 \
  2>&1 | tee "$OUT/flywire_optic_lobe_flow_medium_3seed.log"
```

For a quick AWS smoke run before launching the full optic lobe:

```bash
python scripts/run_optic_flow_benchmark.py \
  --mode all \
  --output-dir "$OUT" \
  --cache-dir "$OUT" \
  --flywire-download-dir /home/ec2-user/pathintegrationBPU/flywire_cache \
  --device cuda \
  --difficulty easy \
  --max-neurons 5000 \
  --models optic_lobe_seeded shuffled_topology random_sparse \
  --seeds 0 \
  --epochs 3 \
  --batch-size 16 \
  --train-batches 4 \
  --val-batches 2 \
  --test-batches 2 \
  --log-every-seconds 10
```
